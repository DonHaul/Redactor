Custom Rules

You can set custom rules in order to redact ignore or replace a certain pattern or string.
There must be only 1 rule per line.
They must have the folowing format:

ACTION METHOD OBJECT

ACTION can be:

  • ignore - it will ignore something
  • redact - it will redact something
  • replace - it will replace something with another thing

METHOD can be:

  • string - looks for a string to ignore/redact/replace
  • regexp - looks for a regular expression to ignore/redact/replace. Look more into regular expressions here carefull though, once you use regular expressions there is no going back :p
  • jquerysel - looks for jquery selectors to ignore/redact/replace. See how to use them here and here, they allow you to select elements based on their id, class or dom place.

Usefull Uses - Hiding your phone number / credit card number

Imagine you want to show a page with a credit cart number or your personal number, and you want to hide that, well just add two rules:

ignore regexp .*
redact regexp [0-9]

the first rule will make it so, nothing is redacted by default

the first rule will look for any numbers on the page and redact them

Examples

  • ignore string john

    Will ignore the word john, and it will not redact it

  • redact regexp (badword|sillyword)

    Will redact the word john, and it will not redact it

  • replace regexp (cat|dog)->turtle

    Will replace the word cat or the word dog, with the word turtle

  • replace string female->male

    Will replace the word female by the word male
